# Geographic Trends in US Kidney Disease Mortality (2017)

A presentation-ready spatial visualization analyzing the age-adjusted mortality rates of kidney disease across the United States. This project implements a professional **Albers Equal Area Conic projection** to accurately represent US geography on widescreen slide decks.

## Final Visual Output
Below is the publication-quality choropleth map generated by this project:

![US Kidney Disease Mortality Map](Kidney%20Disease%20Map.png)

---

## Data Profile & Scope

* **Temporal Scope:** Calendar year **2017** (extracted from the comprehensive 1999–2017 CDC/NVSS historical series).
* **Metric Focused:** Age-adjusted death rate per 100,000 standard population (`DeathRate`).
* **Geographic Layer:** US Census Bureau 2018 State Boundaries (500k resolution scale).
* **Key Visual Insight:** The map highlights a clear regional disparity, showing higher age-adjusted mortality concentrations traveling through the Southern United States, contrasted against lower rates in the West and parts of New England.

---

## Technical Features

* **Advanced Cartographic Projection:** Uses **EPSG:5070** (Albers Equal Area Conic) to eliminate the "flat/stretched" distortion of unprojected GPS coordinates, providing a beautiful, professional curvature.
* **Widescreen Slide Optimization:** Uses `theme_void()` to completely strip background axis grids and coordinates, relocating a wide continuous color legend to the bottom to maximize horizontal real estate for presentations.
* **Perceptually Uniform Color Palettes:** Employs the high-contrast `plasma` color engine from the `viridis` library, ensuring readability for colorblind viewers and sharp translation across projectors.

How to Run This Project
1. Prerequisites
Ensure you have the following packages installed in your R environment:

R
install.packages(c("tidyverse", "sf", "readxl", "viridis"))
2. Line-by-Line Execution Workflow
The script KidneyDisease.R is designed to be executed incrementally line-by-line rather than running a single combined block. This granular approach ensures environment control, strict data verification, and careful memory tracking at every phase:

Individual Library Loading: Packages are loaded one by one (tidyverse, then sf, then readxl, then viridis) to strictly monitor namespace loading.

Step-by-Step File I/O: The boundary shapefile and the Excel data sheet (mortality.xlsx) are read into memory independently to isolate loading times.

Albers Coordinate Transformation: The geometry layout is actively converted to the curved Albers Equal Area Conic projection layout before data integration.

Isolated Data Combination: The relational data frames are carefully joined together via left_join and spatial attributes are mutated into numeric fields while watching for type conflicts.

Canvas Display Rendering: The final ggplot2 engine is executed to draw state borders, assign the Viridis Plasma color spectrum, and pop open the graphical presentation map frame.
